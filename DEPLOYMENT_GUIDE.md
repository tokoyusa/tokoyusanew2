# 📘 PANDUAN LENGKAP DEPLOYMENT - DIGITAL STORE PRO

Ikuti langkah-langkah di bawah ini secara urut untuk menjalankan aplikasi Anda di server produksi (Vercel).

## 1. Persiapan Akun
Pastikan Anda sudah memiliki akun di layanan berikut:
*   **GitHub**: Untuk menyimpan kode sumber. [Daftar di sini](https://github.com/)
*   **Supabase**: Untuk Database & Antarmuka Auth. [Daftar di sini](https://supabase.com/)
*   **Vercel**: Untuk Hosting aplikasi. [Daftar di sini](https://vercel.com/)
*   **Pakasir**: Untuk Payment Gateway (Opsional jika ingin otomatis). [Daftar di sini](https://pakasir.com/)

---

## 2. Setup Database Supabase (PENTING!)
Aplikasi ini tidak akan jalan tanpa database yang benar.
1.  Buat **New Project** di Supabase.
2.  Buka menu **SQL Editor** di sidebar kiri.
3.  Klik **New Query**.
4.  Buka file `FULL_SCHEMA.sql` yang ada di root project ini, salin semua isinya.
5.  Tempel (Paste) ke SQL Editor Supabase, lalu klik **RUN**.
6.  **Hasil**: Semua tabel, kebijakan keamanan (RLS), dan trigger profil otomatis akan dibuat.

---

## 3. Cara Menentukan Admin (OTOMATIS)
Project ini sudah dilengkapi fitur **Auto-Bootstrap Admin**:
1.  Setelah Anda menjalankan SQL Schema di Supabase, silakan buka web Anda.
2.  Lakukan **Daftar/Sign Up** segera.
3.  **PENTING**: Orang pertama yang mendaftar di database tersebut akan **OTOMATIS** mendapatkan role `admin`.
4.  Jika Anda ingin menambah admin lain:
    *   Minta mereka mendaftar.
    *   Buka Tabel `profiles` di Supabase, ubah `role` mereka menjadi `admin` secara manual.

---

## 4. Persiapan GitHub (Import Existing Files)
1.  Buka GitHub, buat **New Repository**. Beri nama (misal: `digital-store-pro`).
2.  Jika Anda menggunakan AI Studio: Klik menu **Settings** > **Export code to GitHub** (Jika tersedia).
3.  Atau secara manual: Upload semua file project ini ke repository GitHub tersebut.

---

## 5. Deployment ke Vercel
1.  Buka Dashboard Vercel, klik **Add New** > **Project**.
2.  Pilih repository GitHub yang baru saja Anda buat, klik **Import**.
3.  **Framework Preset**: Biarkan "Vite" atau "Other" (Vercel akan mendeteksi otomatis).
4.  **Root Directory**: `./` (Root).

### ⚙️ Konfigurasi Environment Variables (KRUSIAL)
Sebelum klik "Deploy", buka bagian **Environment Variables** dan masukkan key berikut satu per satu:

| Key | Value | Catatan |
| :--- | :--- | :--- |
| `VITE_SUPABASE_URL` | https://xxxx.supabase.co | Ambil di Supabase: Project Settings > API |
| `VITE_SUPABASE_ANON_KEY` | (String panjang) | Ambil di Supabase: Project Settings > API (anon public) |
| `SUPABASE_SERVICE_ROLE` | (String sangat panjang) | Ambil di Supabase: Settings > API (service_role secret) |
| `NODE_ENV` | `production` | Wajib untuk mode server |

> **Catatan**: Pastikan file `vercel.json` ada di root project Anda agar API backend bisa berjalan di Vercel.

5. Klik **Deploy**. Tunggu 1-2 menit sampai selesai.

---

## 6. Konfigurasi Webhook & API Pakasir
Setelah web hidup, Anda perlu menyambungkannya agar pembayaran otomatis (Pakasir) bekerja:
1.  Login ke **Admin Panel** aplikasi Anda.
2.  Masuk ke menu **Pengaturan** (Settings).
3.  Input **Pakasir Project Slug** dan **Pakasir API Key**. (Sekarang bisa diatur di Admin saja, tidak perlu di Vercel env).
4.  **Webhook**: Di sana akan muncul link Webhook (misal: `https://web-anda.vercel.app/api/webhook/pakasir`).
5.  Buka Dashboard Pakasir, cari bagian **Integrasi/Webhook**, lampirkan link tersebut.

---

## 7. Troubleshooting
*   **Halaman Kosong?**: Periksa Console log di Browser, kemungkinan besar Supabase URL/Key salah ketik.
*   **Gagal Narik Saldo?**: Pastikan Anda sudah menjalankan SQL Schema karena fitur penarikan saldo butuh tabel `withdrawals`.
*   **Status Pesanan Tetap Pending?**: Cek apakah Webhook URL di Pakasir sudah benar dan mengarah ke link Vercel Anda.
